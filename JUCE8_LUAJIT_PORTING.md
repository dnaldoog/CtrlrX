# JUCE 8 + LuaJIT Porting Guide for CtrlrX

This document describes the steps taken to port CtrlrX to JUCE 8 with LuaJIT support.
The result is shown in the About window:

```
JUCE 8.0.12 | LuaJIT 2.1.1772148810 | LuaBind 0.9.0 | Boost 1.88.0
```

For full LuaJIT build instructions see [LUAJIT_BUILD.md](LUAJIT_BUILD.md).

---

## Overview

The porting process involves:
1. Adding LuaJIT to the project
2. Updating `CMakeLists.txt` for JUCE 8 and LuaJIT
3. Fixing JUCE 8 API breaking changes in source files
4. Building and verifying

---

## Step 1 — Add LuaJIT to Resources

The LuaJIT source is distributed as a zip file. Add it to the repository at:

```
Source/Resources/luajit.zip
```

Reference: https://github.com/dnaldoog/CtrlrX/blob/5.6.35-Maintenance_enhanceII/Source/Resources/luajit.zip

---

## Step 2 — Extract LuaJIT

From the project root, extract the zip to `Source/Misc/`:

**Windows (CMD):**
```cmd
tar -xf "Source/Resources/luajit.zip" -C "Source/Misc/"
```

**Windows (PowerShell):**
```powershell
Expand-Archive -Path "Source\Resources\luajit.zip" -DestinationPath "Source\Misc\"
```

**Linux/macOS:**
```bash
unzip Source/Resources/luajit.zip -d Source/Misc/
```

This creates `Source/Misc/luajit/` with all source files, a pre-configured `CMakeLists.txt` and `.gitignore`.

---

## Step 3 — Update CMakeLists.txt

The main `CMakeLists.txt` needs several changes for JUCE 8 + LuaJIT support.

Reference: https://github.com/dnaldoog/CtrlrX/blob/5.6.35-Maintenance_enhanceII/CMakeLists.txt

### Add LuaJIT option
```cmake
option(CTRLRX_USE_LUAJIT "Build with LuaJIT instead of standard Lua" OFF)
```

### Add LuaJIT subdirectory conditionally
```cmake
if(CTRLRX_USE_LUAJIT)
    add_subdirectory(Source/Misc/luajit)
endif()
```

### ⚠️ Critical — Add to target_compile_definitions
```cmake
target_compile_definitions(SharedCode INTERFACE
    ...
    JUCE_USE_CUSTOM_PLUGIN_STANDALONE_APP=1   ← required for JUCE 8 Standalone
    JUCE_MODAL_LOOPS_PERMITTED=1              ← required for modal dialogs
    $<$<BOOL:${CTRLRX_USE_LUAJIT}>:CTRLRX_USE_LUAJIT=1>  ← LuaJIT flag
    ...
)
```

Without `JUCE_USE_CUSTOM_PLUGIN_STANDALONE_APP=1` and `JUCE_MODAL_LOOPS_PERMITTED=1` the build will fail with errors about missing `runModalLoop`.

### Add conditional LuaJIT linking
```cmake
if(CTRLRX_USE_LUAJIT)
    if(WIN32)
        target_link_libraries("${PROJECT_NAME}" PRIVATE SharedCode
            "${CMAKE_CURRENT_SOURCE_DIR}/Source/Misc/luajit/src/lua51.lib")
    elseif(APPLE)
        target_link_libraries("${PROJECT_NAME}" PRIVATE SharedCode
            "${CMAKE_CURRENT_SOURCE_DIR}/Source/Misc/luajit/src/libluajit.a")
    else()
        target_link_libraries("${PROJECT_NAME}" PRIVATE SharedCode libluajit)
    endif()
else()
    target_link_libraries("${PROJECT_NAME}" PRIVATE SharedCode)
endif()
```

## Step 4 — Build LuaJIT (Windows)

See [LUAJIT_BUILD.md](LUAJIT_BUILD.md) for full instructions.

Quick summary — from **x64 Native Tools Command Prompt for VS 2022**:

```cmd
cd Source\Misc\luajit\src
msvcbuild.bat static
```

Verify x64:
```cmd
dumpbin /headers lua51.lib | findstr machine
```
Should say `8664 machine (x64)`.

## Step 5 — Commit LuaJIT Files

```cmd
git add Source\Misc\luajit\src\lua51.lib
git add Source\Misc\luajit\src\luajit.h
git commit -m "Add LuaJIT lua51.lib and generated luajit.h"
git push origin
```

> `luajit.h` is generated by `msvcbuild.bat` and must be committed — without it
> the build fails with `C1083: Cannot open include file: 'luajit.h'`

## Step 6 — Fix JUCE 8 API Breaking Changes

### 6a — LMemory.cpp and LMemoryBlock.h

Reference:
- https://github.com/dnaldoog/CtrlrX/blob/5.6.35-Maintenance_enhanceII/Source/Lua/JuceClasses/LMemory.cpp
- https://github.com/dnaldoog/CtrlrX/blob/5.6.35-Maintenance_enhanceII/Source/Lua/JuceClasses/LMemoryBlock.h

#### `Thread::setPriority` — enum instead of int

JUCE 8 changed `Thread::setPriority(int)` to `Thread::setPriority(Thread::Priority)`.
The enum values are: `background(-2)`, `low(-1)`, `normal(0)`, `high(1)`, `highest(2)`.
Note: `lowest` does **not** exist — use `background` instead.

```cpp
bool LThread::setPriority(int priority)
{
#if JUCE_MAJOR_VERSION >= 8
    auto p = Thread::Priority::normal;
    if      (priority <= 1) p = Thread::Priority::background;
    else if (priority <= 3) p = Thread::Priority::low;
    else if (priority <= 5) p = Thread::Priority::normal;
    else if (priority <= 8) p = Thread::Priority::high;
    else                    p = Thread::Priority::highest;
    return Thread::setPriority(p);
#else
    return Thread::setPriority(priority);
#endif
}
```

#### `Thread::startThread` — enum instead of int

```cpp
void LThread::startThread(int priority)
{
#if JUCE_MAJOR_VERSION >= 8
    auto p = Thread::Priority::normal;
    if      (priority <= 1) p = Thread::Priority::background;
    else if (priority <= 3) p = Thread::Priority::low;
    else if (priority <= 5) p = Thread::Priority::normal;
    else if (priority <= 8) p = Thread::Priority::high;
    else                    p = Thread::Priority::highest;
    Thread::startThread(p);
#else
    Thread::startThread(priority);
#endif
}
```

#### `Thread::getThreadId()` — cast changed

```cpp
int LThread::getThreadId() const noexcept
{
    return (int)(intptr_t)Thread::getThreadId();
}
```

#### `BigInteger::setBit` — return type changed from `void` to `BigInteger&`

```cpp
// JUCE 8: return type is now BigInteger& not void
.def("setBit", (BigInteger& (BigInteger::*)(int))&BigInteger::setBit)
.def("setBit", (BigInteger& (BigInteger::*)(int,bool))&BigInteger::setBit)
```

### 6b — CtrlrAbout.cpp — Optional LuaJIT version display

Reference: https://github.com/dnaldoog/CtrlrX/blob/5.6.35-Maintenance_enhanceII/Source/UIComponents/CtrlrAbout.cpp

Add conditional include at the top:

```cpp
extern "C" {
#include "lua.h"
#include "lualib.h"
#include "lauxlib.h"
#ifdef CTRLRX_USE_LUAJIT
#include "luajit.h"
#endif
}
```

Add conditional version string:

```cpp
#ifdef LUAJIT_VERSION
    String luaVersion = String(LUAJIT_VERSION);
#else
    String luaVersion = LUA_RELEASE;
#endif
```

## Step 7 — Build

From **x64 Native Tools Command Prompt for VS 2022**:

```cmd
rd /s /q build
cmake -B build -G Ninja ^
  -DCMAKE_C_COMPILER=cl ^
  -DCMAKE_CXX_COMPILER=cl ^
  -DCMAKE_BUILD_TYPE=Release ^
  -DCTRLRX_USE_LUAJIT=ON
cmake --build build
```

> **Important:** Always use the x64 Native Tools Command Prompt.
> Running from regular CMD or PowerShell will use MinGW/GCC and fail with
> `#error "MinGW is not supported"`.

---

## Step 8 — Incremental Builds

Once configured, subsequent builds only need:

```cmd
cmake --build build
```

If build files are locked by a previous process:

```cmd
taskkill /f /im ninja.exe >nul 2>&1
taskkill /f /im cl.exe >nul 2>&1
taskkill /f /im CtrlrX.exe >nul 2>&1
rd /s /q build
```

Then repeat Step 7.

## Verifying the Result

Launch CtrlrX and open **Help → About**. The library version line should show:

```
JUCE 8.0.12 | LuaJIT 2.1.ROLLING | LuaBind 0.9.0 | Boost 1.88.0
```

In the Lua console:

```lua
if jit then
    console(tostring(jit.version))   -- "LuaJIT 2.1.ROLLING"
    console(tostring(jit.status()))  -- true = JIT active
else
    console("Standard Lua 5.1 - no JIT")
end
```

## Summary of JUCE 8 Breaking Changes Fixed

| File | Change | Details |
|---|---|---|
| `CMakeLists.txt` | Added `JUCE_MODAL_LOOPS_PERMITTED=1` | `runModalLoop` requires this in JUCE 8 |
| `CMakeLists.txt` | Added `JUCE_USE_CUSTOM_PLUGIN_STANDALONE_APP=1` | Required for Standalone target |
| `LMemory.cpp` | `Thread::setPriority(int)` → enum | `lowest` renamed to `background` |
| `LMemory.cpp` | `Thread::startThread(int)` → enum | Same Priority enum |
| `LMemory.cpp` | `Thread::getThreadId()` cast | Returns `ThreadID` not `void*` |
| `LMemory.cpp` | `BigInteger::setBit` return type | `void` → `BigInteger&` |
| `CtrlrAbout.cpp` | LuaJIT version display | Conditional `#ifdef CTRLRX_USE_LUAJIT` |
